# Workflow: Build Instructional Module

This workflow guides the creation of a new learning module (e.g., "The Local Forge", "Logic", "Implementation").

## 1. Setup & Context
1. **Read the Storyboard**: Identify the scenes and learning objectives from the [Project Plan](../README.md).
2. **Define the Metaphor**: What is the "Cyberpunk/Sovereign" analogy for this lesson? (e.g., "The Forge", "The Bridge").

## 2. Content Generation
// turbo
3. **Draft Content**: Create the `README.md` for the module using the **Pedagogical Framework**.
4. **Scaffold Prompts**: Create a "recipes.md" file containing the specific AI prompts students will use.

## 3. Technical Implementation
// turbo
5. **Generate Assets**: Create any necessary code snippets or config files (e.g., `Modelfile` templates).
6. **Verify Style**: Ensure all markdown and code follows the **Matrix-Rust** guide.

## 4. Review
7. **Pedagogy Check**: Does this module reduce cognitive load? Is the "Air-Gap" mandated respected?
8. **Visual Check**: Are the headers `VT323`? Is the tone "Sovereign" (empowering, not preachy)?

## Usage
Run this workflow by checking off items as you build `module-X-name/`.
